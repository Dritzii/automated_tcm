Test Closure Memo

This document explains the various activities performed by Test IAC as part of pipeline for the application.


Testing Scope

  • Out of Scope: Performance Testing was not done for this application.
  • Out of Scope: Verification of connectivity with the third party systems, as the connectivity could not be established due to some technical limitations such as access or permissions.
    This can be verified during UAT (User Acceptance Testing) where the connectivity is available or can be established via manual testing or creating web scrapping testing cases where manual button presses become automated.
  • In Scope: Functional Testing for the following modules are in Scope of Testing [Lambda, ECS, EC2, Lambda Security Baseline]
    Any other avaliable Testing components can be customized per Application by updating the code base and injecting logic as per app.
  • In Scope: Any returned data from a developer must adhere and be prepared to create own testing akin to ADO as this Pipeline does not intergrate with ADO at this time.
  • In Scope: Any feedback to improve reporting or units of test are to be emailed to the solutions lead.
  • For more information regarding the above, please contact the solutions group.
